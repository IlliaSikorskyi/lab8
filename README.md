Опис виконаних завдань
8.1 - Stream Readable
Реалізовано зчитування файлів з локальної файлової системи за допомогою fs.createReadStream та передачу вмісту безпосередньо у HTTP-відповідь.

8.2 - Stream Writable
Реалізовано прийом даних через POST-запит та їх запис у файл за допомогою fs.createWriteStream, що дозволяє обробляти великі обсяги даних без перевантаження пам'яті.

8.3 - Stream Transform
Використано потік трансформації (Transform) для обробки даних «на льоту». Реалізовано перетворення тексту файлу у верхній регістр (uppercase) перед відправкою клієнту.

8.4 - Stream Counter
Реалізовано підрахунок кількості байтів та "шматків" (chunks) даних у тілі вхідного POST-запиту з використанням подій .on('data') та .on('end').

8.5 - Stream Errors
Реалізовано безпечну обробку помилок при роботі з потоками. Сервер коректно обробляє запити до неіснуючих файлів, повертаючи статус 500 та повідомлення про помилку, при цьому зберігаючи працездатність процесу.

Технічні особливості
Pipeline: У всіх завданнях для роботи з потоками використовувався метод pipeline (з модуля stream) або перевірки fs.promises, що забезпечує автоматичне керування ресурсами та обробку помилок.

Стабільність: Сервер спроєктовано таким чином, щоб він не завершував свою роботу при виникненні помилок читання файлів.

Як запустити
Для запуску конкретного завдання скористайтеся командою:

Bash
node <назва_файлу>.js 3000
